WHAT WE BUILT
The MVP in one minute
A guided, fully-validated pizza ordering flow that prices every order deterministically and persists it in a parseable log — no manual math, no crashes, every order captured.


8 / 8
Edge cases handled 
Zero unhandled crashes


100%
Bills computed by one deterministic engine


All
Order appended to a parseable log


0
Code changes to swap the menu files

How it works under the hood
Menu loaded at runtime from 3 .txt files (Base · Pizza · Topping) — defensively parsed, never hardcoded.
Pricing engine runs 6 fixed steps: unit price → subtotal → 10% discount at qty ≥ 5 → 18% GST on the post-discount total → payable.
Completed orders are appended to orders_log.txt, pipe-separated, one block each — ready for AOV, peak-hour & break-even analysis.

WHY THE NUMBERS ARE TRUSTWORTHY
Pricing engine — the worked example, verified
One deterministic path, six steps, same result every time. This is the client's reference order — the app reproduces it to the paisa.

Base — Cheese Burst
₹229.00
Pizza — BBQ Chicken
₹379.00

Topping — Extra Cheese
₹69.00
Unit price
₹677.00

Subtotal (× qty 5)
₹3,385.00
Discount — 10% (qty ≥ 5)
− ₹338.50

Taxable amount
₹3,046.50
GST @ 18%
₹548.37

Total payable
₹3,594.87
THE SIX STEPS

1
Unit price = Base + Pizza + Topping

2
Subtotal = Unit price × Quantity

3
Discount = 10% if Qty ≥ 5, else ₹0

4
Taxable = Subtotal − Discount

5
GST = 18% of Taxable amount

6
Total = Taxable + GST
